## 1\) Scope

## 2\) Data We Process

### A) Data you choose to submit

**Customer Content (your ToCs):** graph structures, nodes, assumptions, evidence links, and labels.

**Support/feedback:** messages you send us (e.g., by email or forms). We may keep these to respond and improve the Service. We only record calls with clear notice and your agreement.

### B) Data we collect automatically (minimal)

**Technical logs:** IP address, date/time, requested URLs, device/OS/browser, referrer. Generated by Cloudflare (which provides both DNS and our hosting runtime, Cloudflare Workers) and our server code. Used for security, abuse prevention, routing, debugging, and to keep the Service reliable.

**Anonymous-visitor identifier (anti-abuse):** To enforce a per-person free-tier spend cap of $5 without requiring sign-in, we mint a random UUID once per browser on your first visit and persist it in an HttpOnly first-party cookie (`tocb_actor_id`, Max-Age 1 year). The UUID is generated by the server's cryptographic random source and is not derived from your IP address or any other personal data. This identifier is the key we use to attribute free-tier AI spend to the same visitor across requests. Each browser gets its own identifier, so two people sharing the same network (office NAT, CGNAT, campus wifi) get independent caps. See Section 2G for the associated spend data.

**Auth-link cookie (cap attribution across sign-out):** If you sign in, we also set an HttpOnly first-party cookie named `tocb_auth_link` (Max-Age 1 year, HMAC-signed with a server-side secret) that records which Auth0 user ID this browser has been signed in as. If you then sign out and continue as an anonymous visitor, we read this cookie and keep attributing your AI spend to your authenticated cap row, instead of letting you reset the $5 cap by logging out. The cookie is consulted only for cap attribution; it is not used for permission checks, BYOK access, or anywhere else a JWT is required.

**Anti-abuse challenge:** For anonymous visitors we use Cloudflare Turnstile (a CAPTCHA alternative) to prevent automated abuse of free-tier AI calls. On a successful challenge we set a third, short-lived HttpOnly cookie (`tocb_anon`, Max-Age 24 hours) that simply proves you solved the widget; it does NOT hold or affect the spend identifier. When it expires we'll ask you to re-solve. Turnstile may process limited device/browser signals under Cloudflare's own terms; we only receive a pass/fail verification token.

### C) Account data (if you sign in)

When you authenticate via Auth0, our authentication provider stores a user profile on our behalf, including your user ID, email address, and name (which may default to your email address). Auth0 also records login metadata such as IP addresses, login timestamps, and login counts as part of its normal operation.

We store your user ID and email address in our own database for access control and to show who has access to shared charts.

### D) AI Improvement Data

We collect usage data to evaluate and improve our AI assistant, unless you opt out. This includes:

- **Session metadata:** A randomly generated session ID, chart ID, your user ID (if logged in), browser user agent, and timestamps.  
- **Chat messages:** The prompts you send to the AI and the AI's responses, including token usage counts. Messages are stored in association with the chart they relate to. If logged in, your user ID is attached.  
- **Graph snapshots:** The state of your graph after each edit (manual, AI, or undo/redo). Snapshots are stored in temporal sequence, enabling reconstruction of the editing history. For AI edits, we also store the edit instructions, whether the edit succeeded, any error details, and a link to the specific chat message that triggered the edit. If logged in, your user ID is attached.
- **Error reports:** When an error occurs during AI interactions, we log the error type and message, HTTP status code, a truncated stack trace (up to 4 KB), your browser user agent, and request metadata (such as the AI model used and features enabled). If logged in, your user ID is attached; the relevant chart ID and session ID are included when available.

Sessions are identified by random UUIDs, and users by opaque Auth0 user IDs. **No email addresses or names are stored** in the AI improvement logging tables.

**What we do NOT collect if you opt out:** If you disable usage data sharing in the privacy popup or in your account privacy settings, we do not store any session, message, or snapshot data. When you opt out, usage data collection stops immediately. Your opt-out preference is enforced both in your browser and on our servers; the only server communication is to record your preference and clean up any existing session. When you are logged in, your preference persists across devices and browsers.

**No marketing/ads analytics.** We do not use tracking for advertising.

### E) File uploads (Chat and Generate modes)

When you attach a file (such as a PDF) in the Chat or Generate interfaces, the file is uploaded to Anthropic's Files API and stored there until you remove it (by clearing the chat or deleting the chart). Anthropic does not use uploaded files to train their AI models (per Anthropic's Commercial Terms). Files persist at Anthropic until you explicitly delete them; messages referencing those files follow Anthropic's default 7-day API retention.

We keep a local record of each uploaded file (the Anthropic file ID, the chart it belongs to, the original filename, and the upload timestamp) so that we can list your files in the UI and delete them on your behalf. Deleting the chart, pressing "Clear Chat", or submitting a data erasure request (see Section 8) removes both the local record and the file at Anthropic.

### F) Bring Your Own Key (optional)

You may optionally supply your own Anthropic API key ("Bring Your Own Key") to bypass the free-tier spending cap or to use Generate mode. When you do, your key is encrypted at rest on our servers using AES-GCM with a single server-side master key, bound to your Auth0 user ID via AAD (additional authenticated data), so the ciphertext for one user cannot be decrypted under another user's ID. We only use the decrypted key to forward your requests to Anthropic. You can delete your key at any time from the settings panel. If you delete your key, your encrypted blob is zeroed and removed from our database.

We do not log the plaintext key, and we do not use the key for any purpose other than forwarding the request you initiate. Requests made with your key are still subject to Anthropic's Commercial Terms and privacy policy.

### G) Cost and spend accounting

To run a free tier responsibly we track cumulative AI spend per user on our servers. This is a small numerical record (no message content), and exists in two places:

- **Per-person cumulative spend** (`user_api_usage` table): the total micro-USD we've spent on Anthropic API calls on your behalf, plus per-feature token counters (input, output, cache reads/writes, web-search uses). Keyed to your Auth0 user ID when signed in, or to the anonymous identifier described in Section 2B otherwise. Used to enforce the free-tier spend cap and to display the "$X of $5 used" indicator. BYOK requests do not accumulate here (you're billed directly by Anthropic).
- **Global monthly spend** (`global_monthly_usage` table): a single aggregate total per calendar month across all users, used to honour Anthropic's per-workspace console-side cap. No per-user fields.

Counters reset under the conditions described in Section 6 (Retention).

## 4\) How We Share Data

We do not sell personal data. We share only as needed to run the Service:

- **Hosting:** Cloudflare Workers (Cloudflare, Inc.) runs the Service; both the static assets and the server code are served from one Worker. Cloudflare processes IP addresses and request metadata as part of routing, protecting, and executing traffic.
- **Database:** Neon (Neon Inc., serverless PostgreSQL) stores chart data, AI improvement logging data, and the cost-accounting records described in Section 2G.
- **Anti-abuse:** Cloudflare Turnstile verifies that anonymous visitors are not automated clients. Turnstile's processing is governed by Cloudflare's own terms.
- **Authentication:** Auth0 (Okta, Inc.) stores user profiles (user ID, email, name, login metadata) on our behalf and processes authentication credentials.
- **AI provider:** Anthropic Claude processes AI assistant requests. When you interact with the AI assistant, the full content of your Theory of Change graph (including all node text, evidence, assumptions, and connections), your chat message history from the current session, any selected nodes, and any files you have attached are sent to Anthropic to generate a response. Files you attach are stored at Anthropic via the Files API (see Section 2E) until you remove them. Anthropic's paid API terms state that API data is not used for model training by default. We do not opt in to any data-sharing or training programs. However, Anthropic may retain data temporarily for abuse monitoring and safety review, and its policies may change.
- **Compliance & safety:** To competent authorities where required by law or to protect rights, safety, and security.
- **Business changes:** If we undergo a merger, acquisition, or reorganisation, data may transfer under appropriate safeguards.

AI improvement logging records stay in our database (hosted by Neon). They are not sent to AI providers or any other third party. Only the real-time chat prompts transit through Anthropic as part of normal service operation.

## 7\) Security

We aim to be transparent about how we protect your data and the limits of our protections.

**No end-to-end encryption / readable at rest.** Customer Content is not end-to-end encrypted and is stored in a form that our authorised staff (e.g., backend developers) can access. Do not upload highly confidential, secret, or special-category data (e.g., health, political opinions) to the Service.

**When we access Customer Content.** Access by staff is restricted and permitted only:

- (a) **With your direction:** when the owner asks for help (we verify ownership by control of the edit link to the ToC graph or other reasonable proof),  
- (b) **When strictly necessary:** to diagnose/fix an outage, bug, or data integrity issue affecting your content or the Service more broadly, or to investigate abuse/security incidents, or  
- (c) **Where required by law.**

**AI Improvement Data access.** This data may be accessed by our team to analyse AI performance, debug issues, and improve the Service. We do not share raw AI improvement data externally except with processors under appropriate agreements.

**How we control access.** We apply role-based access; least-privilege permissions; required multi-factor authentication for staff accounts.

**Hosting and dependencies.** The Service is hosted on Cloudflare Workers. Chart data, AI improvement data, and cost-accounting data are stored in Neon (PostgreSQL). Operational security also includes routine patching, vulnerability management, and change control.

**Breach response.** If we become aware of unauthorised access, loss, or disclosure of personal data, we will investigate, mitigate, and notify affected users and regulators when legally required. If needed, we may rotate or invalidate edit links.

**Your responsibilities.** Treat edit links as sensitive (they act like keys). Share them only with people you trust. Avoid uploading confidential or special-category personal data. Be cautious with links and attachments you include in your ToC.

**Report a vulnerability or security concern.** Email [theoryofchangebuilder@gmail.com](mailto:theoryofchangebuilder@gmail.com) with details. We appreciate responsible disclosure. We do not currently operate a paid bug-bounty programme.

## 10\) Cookies & Local Storage

We use browser storage for functional purposes.

**Cookies (set by our server):**

- `tocb_actor_id`: HttpOnly, Secure, SameSite=Lax. Max-Age 1 year. Stores a random UUID minted once per browser, as described in Section 2B. This is the key we use to attribute free-tier AI spend to the same visitor across requests. Not derived from your IP or any other personal data.
- `tocb_auth_link`: HttpOnly, Secure, SameSite=Lax. Max-Age 1 year. HMAC-signed with a server-side secret; payload records the Auth0 user ID this browser has been signed in as. Consulted only when you are signed out, to keep attributing your AI spend to your authenticated cap row (prevents a "log out to reset the cap" loophole). Refreshed on every successful sign-in.
- `tocb_anon`: HttpOnly, Secure, SameSite=Lax. Max-Age 24 hours. Proves you solved a Cloudflare Turnstile challenge. Independent of the spend identifier; when this expires we ask you to re-solve the widget, but your spend counter is unaffected.

**Local storage (stays on your device):**

- **Authentication:** Login tokens and return-path after authentication
- **Privacy preferences:** Privacy policy acceptance status, usage data sharing preference
- **Logging session:** Session identifiers for active usage logging sessions
- **Chat history:** Your AI assistant conversation history, stored per chart so messages persist across page reloads
- **Graph backups:** Local copies of your chart data for recovery purposes
- **User settings:** Custom AI system prompts, recently edited chart metadata, tutorial completion status
- **BYOK spend counters:** If you use Bring Your Own Key, a running total of spend per chart and per key last-4 digits. Stored locally only; cleared when you remove the key or delete the chart.

Local-storage data is not transmitted to our servers except where described in this policy (e.g., chat messages sent to the AI provider during use). You can clear this data at any time through your browser settings or the "Clear Chat" button in the AI assistant.

We do not use marketing or advertising cookies.

## Annex A: AI Features

The Service includes an AI assistant powered by Anthropic Claude that helps users build and refine Theory of Change diagrams. The assistant can guide users through diagram creation, suggest improvements, challenge assumptions, and answer questions about your ToC. AI features are optional; the core editor works without them.

**What we send to Anthropic:** When you interact with the AI assistant, we send your message, the full content of your Theory of Change graph (including all node text, evidence, assumptions, and connections), your chat history from the current session, any nodes you have selected, any files you have attached (uploaded separately to Anthropic's Files API), and system instructions that help the AI understand the graph format.

**Purpose:** To generate the AI response you requested and maintain service quality (latency, error rate, abuse prevention).

**Vendor:** Anthropic Claude API. Anthropic's paid API terms state that API data is not used for model training by default. We do not opt in to any training or data-sharing programs offered by Anthropic. However, Anthropic may retain data temporarily for abuse monitoring, and its policies may change. We recommend reviewing the current terms for the most up-to-date information.

**Retention at Anthropic:** Messages API content (prompts and responses) is retained for 7 days at Anthropic (reduced from 30 days on 14 September 2025). Files you upload via the Files API are retained until you delete the file, delete the chart, or submit a data erasure request. Flagged content (trust-and-safety review) may be retained up to 2 years, and classifier scores up to 7 years, per Anthropic's policies.

**Legal bases:** Contract performance / legitimate interests.

### AI Improvement Data Collection

Separately from real-time AI features, we collect usage data to improve the AI, unless you opt out:

- **What we collect:** Session metadata (session ID, chart ID, user ID if logged in, user agent, timestamps), chat messages (your prompts and AI responses with token counts, linked to the relevant chart), graph snapshots stored in temporal sequence after each edit (graph state, edit instructions for AI edits, success/error status, link to the triggering chat message), and error reports (error type, message, HTTP status, truncated stack trace, browser user agent, request metadata such as the AI model and features enabled).  
- **What we do NOT collect:** Email addresses or names. Sessions are identified by random UUIDs; users by opaque Auth0 user IDs only.  
- **Purpose:** Evaluate AI quality, identify failure patterns, and improve prompt engineering.  
- **Opt-out:** Disable usage data sharing in your account privacy settings. Your preference is enforced both in your browser and on our servers.  
- **Retention:** Up to 24 months, then automatically deleted.  
- **Legal basis:** Legitimate interests, Art. 6(1)(f) GDPR.
